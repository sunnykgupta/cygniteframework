Views

What is Views ?

Views are just web pages where your headers,menus,sidebar,contents and footers resides. Since we follow MVC patterns views neither call directly nor passed any parameters into it, views can be called from controllers.

I have shown you in controllers section how to create controller and now lets create views and load it from controller.

View are reside into your apps/views/controller-name-as-directory/. Based on your controller, cygnite will create view directory where you need to create your view pages. No need to spend more time to create your layout template page, Cygnite has built in functionality to create layout template which reduce your work. You can over right stylesheets and rearrange your dynamic template based upon your needs.

Simple view page

View filename should have prefix .view. Lets create simple view page - 

 <html>
 <head>
       <title>Welcome to Cygnite Framework</title>
 </head>
 <body>
        <h2></h2>
       <div class="content">
             Welcome to Cygnite Framework
        </div>
 </body>
 </html>
You can render your view page by render method, for example- 
   $this->render('welcome')->with(array());
Pass empty array in if you don't want to pass any values into the view page.

How to render Layout Template ?

You can also render layout template and change it's sections based on your needs. Lets us see how to render template by controller. For example-

    $this->createsections(array('header'=>'header.view',
                               'content'=>'register.view',
                               'footer'=>'footer.view'));
    $this->setlayout('admin',array(
                                    'title'=> "Welcome to Cygnite Framework",
                                    'header_title'=> "Header Page",
                                    'content_title'=> "Content Page",
                                    'footer_title'=> "Footer Page"
                                                ));

Here we have rendered admin layout template with its sections. Remember your template page name should have prefix your-template-name.layout.php.

Lets create admin template layout to render it. 

<?php $this->renderlayout($this->template['header']); ?>
<div class="container">
    <div class="container-body" style="margin-left:100px;">
        <?php
          $this->renderlayout($this->template['content']); ?>
    </div>
</div>
<?php $this->renderlayout($this->template['footer']); ?>
We are done ! All your sections will render in corresponding place as above. For more detail example you can also have a look at the template example included with Cygnite home controller.

Now lets call index action via browser url to view output. For example- 

http://www.example.com/cygnite/index.php/home/index

You can also import any helpers into the view page in two different ways, 

  use Cygnite\Helpers\Assets as Assets;
  use Cygnite\Helpers\Url as Url;
and now you can easily access above helpers functions, 
  echo Url::basepath();
  echo Assets::addstyle(' your style sheet url');
Or You can also directly use your functions of helpers as below- 
  echo \Cygnite\Helpers\Assets::addscript('webroot/js/cygnite/jquery.js');
  echo \Cygnite\Helpers\Url::basepath();

How to retrive values into view page ?

You retrive data into the view page as below- 

     echo $this->params['test'];
